## 슬라이드 9: Arbitrum Layer 2 선택 이유

```
┌─────────────────────────────────────────────┐
│                                             │
│     ⚡ 왜 Arbitrum인가?                      │
│                                             │
│                                             │
│  💰 낮은 가스비                              │
│     이더리움 메인넷 대비 1/10~1/100          │
│     → 기부 플랫폼에 적합                     │
│                                             │
│  ⚡ 빠른 속도                                │
│     트랜잭션 확정 ~1초                       │
│     → 사용자 경험 향상                       │
│                                             │
│  🔒 이더리움 보안                            │
│     Rollup 기술로 메인넷 보안 유지           │
│     → 탈중앙화 보장                          │
│                                             │
│  🚀 실전 배포 가능                           │
│     Sepolia(테스트) → One(메인넷) 전환 쉬움  │
│                                             │
└─────────────────────────────────────────────┘
```

**다이어그램**: Layer 2 구조
```
     사용자
       ↓
   Arbitrum (Layer 2)
   - 빠름, 저렴
       ↓
   Ethereum (Layer 1)
   - 보안, 최종 확정
```

## 슬라이드 11: 기부 함수 (코드)

```
┌─────────────────────────────────────────────┐
│                                             │
│    💰 donate() 함수                         │
│                                             │
│  function donate(                           │
│      uint256 campaignId,                    │
│      string message                         │
│  )                                          │
│      external payable                       │
│      nonReentrant    // 🔒 보안             │
│      whenNotPaused   // 🛑 긴급 중지        │
│  {                                          │

│      require(msg.value > 0, "...");         │

│                                             │
│      // 📢 이벤트 발생 (투명성)              │
│      emit DonationMade(...);                │
│  }                                          │
│                                             │
└─────────────────────────────────────────────┘
```

**강조**: 보안 기능 (nonReentrant, whenNotPaused)

## 📝 발표 스크립트 예시

### 슬라이드 1-2 (도입, 30초)
"안녕하세요. 오늘 여러분께 소개할 프로젝트는 '기부 마을'입니다. 
기부를 게임처럼 즐긴다? 이상하게 들리시나요? 
하지만 저희는 이 질문에서 출발해서 블록체인 기반의 
완전히 새로운 기부 플랫폼을 만들었습니다."

### 슬라이드 3-7 (문제와 해결, 2분)
"기부 플랫폼의 가장 큰 문제는 투명성입니다. 
내가 기부한 돈이 어디에 쓰였는지, 정말 제대로 쓰였는지 
알 수 없다는 것이죠.

블록체인은 이 문제를 해결할 수 있습니다. 
무결성, 투명성, 신뢰성 - 이 세 가지 특성으로요.

하지만 단순히 블록체인만 쓴다고 해결되는 게 아닙니다. 
저희는 '게임화'를 통해 재미까지 더했습니다."

### 슬라이드 8-15 (기술 스택, 4분)
"기술적으로 저희는 Arbitrum Layer 2를 사용했습니다. 
왜냐하면 가스비가 저렴하고 속도가 빠르기 때문입니다.

스마트 컨트랙트는 Solidity로 작성했고, 
OpenZeppelin의 검증된 라이브러리를 사용해 보안을 강화했습니다.

특히 주목할 점은 DID와 Verifiable Credential입니다. 
메타마스크 없이도 사용자가 쉽게 접근할 수 있고, 
기부 이력을 검증 가능한 자격증명으로 발급받을 수 있습니다."

### 슬라이드 16-22 (아키텍처와 데모, 3분)
"시스템 아키텍처를 보시면, 프론트엔드는 React로, 
백엔드는 Supabase로, 그리고 핵심 기부 로직은 
블록체인에 있습니다.

실제 화면을 보시죠. 귀여운 마을에서 각 집이 
기부 단체를 나타냅니다. 클릭하면 기부할 수 있고, 
모든 거래는 Arbiscan에서 확인할 수 있습니다."

### 슬라이드 23-27 (향후 계획, 2분)
"향후에는 NFT 아이템 시스템, 메타버스 전환, 
그리고 글로벌 확장을 계획하고 있습니다. 
기부가 진짜 자산이 되고, 사람들이 모여 소통하는 
커뮤니티 플랫폼으로 발전시킬 것입니다."

### 슬라이드 28-30 (마무리, 1분)
"기부 마을은 단순한 기부 플랫폼이 아닙니다. 
블록체인으로 투명성을 보장하고, 
게임화로 재미를 더한 새로운 경험입니다.

투명한 선의, 따뜻한 기부. 
감사합니다."
